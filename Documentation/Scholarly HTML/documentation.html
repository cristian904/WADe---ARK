Abstract

This solution offers an interactive way to explore the Romanian Heritage. Using this platform you can easily browse through different museums and works of art available in Romania. Our main purpose is easing the access to the Works of art in Romanian museums. Besides that, we offer an easy way to visualize data and statistics around museums, artists and pieces of art.

Introduction

The report describes the concepts and technologies that will be applied in the development of the project. We will go through the specific APIs and Web Services that will be present in the project, the general architecture of the project that presents the main components/models, but also a general overview that will incorporate all these concepts into one web application. This projects sets out to offer an intelligent and interactive way of visualizing to visualize relevant and verified data about different works of art available in Romanian museums. The work is being done as part of the Web Application Development course held by Conf. Dr. Sabin-Corneliu Buraga, professor at the Faculty of Computer Science in Iasi.

Web services and APIs

In order to achieve this we will use a set of web services and APIs in order to fetch our data. This will spare us of the trouble of gathering the date regarding the artistic works that we will have to model and display.

DBPedia

DBpedia allows you to ask sophisticated queries against Wikipedia, and to link the different data sets on the Web to Wikipedia data. We will use these capabilities to find out more information about certain artistic work/artist/museum.

Getty Vocabularies

This contain structured terminology for art and other material culture, archival materials, visual surrogates, and bibliographic materials. This can be also used as a source for gathering relevant about artistic work/artist/museum and other data useful to the project.

SPARQL

We will access the data provided via DBPedia and Getty Vocabularies via a SPARQL endpoint. SPARQL is RDF query language, that is, a semantic query language for databases, able to retrieve and manipulate data stored in Resource Description Framework (RDF) format. So this will help us execute complex queries and provides an easy way to handle the data.

Governmental Data

The Romanian government offers, freely, different data sets. We will use the information they have available for statistical use, regarding the Romanian Heritage. This data will point us to the main artists / works of art / museum available in Romanian. The information extracted from here is minimal that’s why we need the help of DBPedia in achieving an exhaustive search.

PROV Book

Now we have all the data we need, from the data providers mentioned above, but we should also check the provenance of each piece of data. Provenance is a record that describes the people, institutions, entities, and activities, involved in producing, influencing, or delivering a piece of data or a thing in the world. So, only after we do this step, it’s ok to display the data on the page.

Architecture

Client

The client will be a web interface developed with Angular 5. We choose Angular because of its strong performance. It can be easily deployed on any web server and it offers a large set of tools that ease the development process. Angular is a client side framework of JavaScript which is based on MVC Architecture and provides a Single-page application solution for dynamic websites. Angular’s strong points are:

  • Code Re-usability
  • Maintenance of the existing code
  • Testing of the code
  • Modularity and Scalibility for Team Development

So, in Angular, code-re usability can be achieved using Modules, Components. It provides the sharing of the code within angular Application. It also Provides ‘Dependency Injection’ (DI) and ‘Angular Routing’ features which provides the re usability of the code again and again. It provides a modular and highly scalable code to enhance the speed of development.

Server

The Spring Framework is an open source application framework that aims to make J2EE development easier. Spring aims to help structure whole applications in a consistent, productive manner, pulling together best-of-breed single-tier frameworks to create a coherent architecture. Spring framework provides flexibility to configure beans in multiple ways such as XML, Annotations, and JavaConfig. With the number of features increased the complexity also gets increased and configuring Spring applications becomes tedious and error-prone.

Database

For persistence we will use PostgreSQL. As the information provided my the Romanian government on heritage is stored in a XML file we will extrac the useful information and store it in a relational database, for easier access and quering.

Overview

So the application, hosted on a web server will query the relational database (obtained from Gov Data XML) for information about Romanian artistic attractions. As the information provided from this query is limited, the client application will call the DBPedia API to further enrich the information, based on the results from the first query. We could also use Getty Vocabularies instead of DBPedia for gathering this information, but it’s not decided yet.

User Workflows

The whole application revolves around Romanian museums and what works of art can be found in them. It can be used in both exploring what can be found and also specific searches about Works of Art, Museums or Artists.

Museums

The main page is represented by the Romanian Map on which all the museums inside The Romanian Heritage dataset can be find. Their information is updated and completed by DBpedia informations. On clicking on each museum figure, further information is available, including a few statistics and a hyperlink to all the Art Works located here.

Works of Art list

Lists with works of art can be accesed in both museum view and All works of art view. They can be filtered by both name and artist's name. On click further information can be achieved. This page also contains hyperlinks to more info about the artists.

Work of Art view

All the information gather from both the gov data and DBpedia is available in this view for a piece of art. Besides that, recommendations are offered for Art in the same time period, same current or artists with resembling style.

Artists

Here, data about artists can be found. Besides pure data, you can access the museums where their works can be find, their paintings and also some statistics.
